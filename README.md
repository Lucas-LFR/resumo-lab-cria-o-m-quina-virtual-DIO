SLA (Service Level Agreement)
Neste módulo, compreendemos que o SLA define os tempos de disponibilidade e que se paga pela rapidez e garantia desse atendimento.

A Regra dos 9: Quanto mais "noves" compõem a porcentagem de disponibilidade (ex: 99,99%), menor é o tempo de indisponibilidade permitido. Por outro lado, quanto menos "noves", maior é o tempo que o serviço pode ficar fora do ar.

Criação de Máquinas Virtuais
Ao configurar máquinas virtuais, aprendemos que as escolhas de infraestrutura impactam diretamente no SLA e nos custos:

Opções de Disponibilidade: Podemos selecionar diferentes níveis de redundância.

Zonas de Disponibilidade: É possível distribuir a carga em até 3 zonas diferentes dentro de uma região para garantir que o serviço não pare caso um data center falhe.

Redundância de Armazenamento: Aprendemos a configurar a replicação de dados em outras regiões. Isso é fundamental para estratégias de Recuperação de Desastres (Disaster Recovery) e para manter a Alta Disponibilidade.
